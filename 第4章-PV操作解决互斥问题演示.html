📌 PV操作逻辑（互斥模型）
• P(mutex) : 申请临界资源 → mutex.value-- ，若 value < 0 则进程阻塞进入等待队列。
• V(mutex) : 释放临界资源 → mutex.value++ ，若 value ≤ 0 则唤醒队列首进程，使其进入临界区。
• 互斥规则 : 任何时刻最多一个进程处于临界区(CR被占用)，其他进程必须阻塞等待。
• 成对匹配 : 进入临界区后必须执行一次V操作释放，演示中只有临界区内进程的V按钮可用。

点击进程「P操作」申请CR，进入临界区后「V操作」释放。若CR被占用，进程自动阻塞；释放时唤醒等待者。
